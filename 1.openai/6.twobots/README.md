# 멀티봇 대화 (Two Bots)

두 GPT 봇이 파일을 공유하며 자동으로 대화하는 패턴을 학습합니다.

## 예제 목록

| 예제 | 파일 | 설명 |
|------|------|------|
| 기본 대화 | `1.debate/bot1.py`, `bot2.py` | 두 봇이 번갈아 대화 (단순 질의응답) |
| 심화 토론 | `2.debate_advanced/bot1.py`, `bot2.py` | 시스템 프롬프트로 역할 설정 (학생↔교수) |

## 아키텍처

```
Bot1 (질문자)       conversation.txt       Bot2 (답변자)
  ──write──→      [공유 파일]       ←──read──
  ←──read───                       ──write──→
```

- 두 봇은 `conversation.txt` 파일을 공유하며 교대로 읽기/쓰기
- 홀수/짝수 줄 수로 자신의 차례를 판단

## 패턴 비교

| 패턴 | 시스템 프롬프트 | 히스토리 활용 | 용도 |
|------|---------------|-------------|------|
| 기본 | 없음 | 마지막 1줄만 | 단순 대화 연습 |
| 심화 | 역할 설정 (학생/교수) | 최근 N개 대화 | 주제 심화 토론 |

## 실행

```bash
# 터미널 2개 열어서 각각 실행
python 1.debate/bot1.py
python 1.debate/bot2.py
```

## 주요 학습 포인트

- **파일 기반 봇 간 통신**: 별도 프로세스의 봇이 파일로 메시지 교환
- **시스템 프롬프트 역할 고정**: system 메시지로 봇의 성격/전문성 제어
- **히스토리 윈도우**: 최근 N개 대화만 참조하여 토큰 절약
