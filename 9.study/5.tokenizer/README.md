# 토크나이저 (Tokenizer)

텍스트를 모델이 이해할 수 있는 토큰(숫자) 단위로 변환하는 핵심 전처리 과정을 학습합니다.

## 토크나이저란?

```
"나는 학생입니다" → ["나는", "학생", "##입니다"] → [2145, 8923, 11234]
   텍스트              토큰(서브워드)              토큰 ID(정수)
```

모든 언어모델은 텍스트를 직접 이해하지 못합니다. 토크나이저가 텍스트를 숫자로 변환해야 모델이 처리할 수 있습니다.

## 주요 토크나이저 알고리즘

| 알고리즘 | 사용 모델 | 원리 | 한국어 특징 |
|---------|----------|------|-----------|
| **WordPiece** | BERT, DistilBERT | 빈도 기반 서브워드 병합 | `##` 접두사로 서브워드 표시 |
| **BPE** (Byte-Pair Encoding) | GPT-2, LLaMA | 가장 빈번한 바이트 쌍을 반복 병합 | 한국어 글자가 여러 토큰으로 분리될 수 있음 |
| **SentencePiece** | T5, EXAONE, Qwen | 언어 무관 유니코드 기반 | `▁`로 공백 표시, 한국어에 유리 |

## 예제

| 파일 | 설명 |
|------|------|
| `1.tokenizer_compare.py` | 3가지 토크나이저 알고리즘 비교 (WordPiece, BPE, SentencePiece) |
| `2.korean_tokenizer.py` | 한국어 토크나이징 심화 — 한영 비교, 토큰 효율성 분석 |
