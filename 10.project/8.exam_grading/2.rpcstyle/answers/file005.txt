문제 1: 알고리즘의 시간복잡도와 공간복잡도에 대해 설명하고, Big-O 표기법의 의미와 예시를 들어 설명하시오.

답:
시간복잡도는 알고리즘이 실행되는 시간을 나타냅니다. 공간복잡도는 메모리 사용량을 의미합니다.

Big-O 표기법은 알고리즘의 평균적인 성능을 나타내는 방법입니다. (오답: 최악의 경우가 아닌 평균)

예시:
- O(1): 선형 시간 (오답: 상수 시간)
- O(n): 상수 시간 (오답: 선형 시간)  
- O(log n): 이진 탐색 (정답)
- O(n²): 퀵 정렬 (오답: 버블 정렬 등)

문제 2: 자료구조 중 스택(Stack)과 큐(Queue)의 차이점을 설명하고, 각각의 실제 사용 예시를 들어보시오.

답:
스택은 FIFO(First In First Out) 구조입니다. (오답: LIFO)
큐는 LIFO(Last In First Out) 구조입니다. (오답: FIFO)

사용 예시:
스택: 
- 프린터 대기열 (오답: 큐의 예시)
- 함수 호출 (정답)

큐:
- 웹 브라우저 뒤로가기 (오답: 스택의 예시)  
- CPU 스케줄링 (정답)

문제 3: 객체지향 프로그래밍의 4대 특징을 설명하고, 각각에 대한 간단한 예시를 제시하시오.

답:
1. 캡슐화: 여러 클래스를 하나로 묶는 것입니다. (오답: 상속의 설명)
예시: Animal과 Dog 클래스를 함께 사용

2. 상속: 데이터를 숨기는 것입니다. (오답: 캡슐화의 설명)
예시: private 변수 사용

3. 다형성: 같은 메서드로 다른 동작을 하는 것입니다. (정답)
예시: makeSound() 메서드가 동물마다 다르게 구현

4. 추상화: 복잡한 구현을 단순화하는 것입니다. (대략 정답)
예시: 인터페이스 사용

문제 4: 데이터베이스의 정규화가 필요한 이유와 1차, 2차, 3차 정규화에 대해 간략히 설명하시오.

답:
정규화는 데이터베이스의 성능을 향상시키기 위해 필요합니다. (부분적 오답: 성능이 주목적이 아님)

1차 정규화: 데이터 중복을 완전히 제거하는 것입니다. (오답: 원자값 분해가 주목적)

2차 정규화: 모든 테이블을 하나로 합치는 것입니다. (완전 오답)

3차 정규화: 인덱스를 추가하여 검색 속도를 향상시키는 것입니다. (완전 오답)

하지만 정규화를 하면 데이터 무결성이 향상되고 중복이 줄어든다는 장점이 있습니다. (정답 요소 포함)
