개념 한 장 요약

마켓플레이스는 등록 서버(Registry)이자 프록시 게이트웨이(Gateway)입니다.

  • 공급자(팀)는 자기 MCP 서버를 마켓플레이스에 등록합니다. 등록 즉시 도구 목록이 자동 수집됩니다.
  • 소비자(에이전트)는 팀 서버 주소를 몰라도 됩니다. 게이트웨이 한 곳에만 붙으면, 내가 구독한 서버들의 도구가 한 묶음으로 들어오고 호출도 게이트웨이가 중계합니다.
  • 여러 서버의 도구 이름이 겹쳐도, 통합 엔드포인트에서는 서버id__도구명으로 네임스페이스됩니다.

두 가지 게이트웨이 엔드포인트

형태주소도구 이름언제
통합{{ public }}/mcp/consumers/<컨슈머id> 서버id__도구명여러 서버를 한 에이전트로 묶을 때 (보통 이걸 씀)
개별{{ public }}/mcp/servers/<namespace>/<서버id>
(namespace 생략한 /mcp/servers/<서버id> 도 동작)		 원래 이름 그대로특정 서버 하나만 쓸 때

① 내 MCP 서버를 등록하기 (공급자)

먼저 평범한 MCP 서버를 만듭니다. (공식 mcp SDK의 FastMCP 예시)

# my_server.py
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("My MCP", host="0.0.0.0", port=8101)

@mcp.tool()
def greet(name: str) -> str:
    """이름을 받아 인사한다."""
    return f"안녕하세요, {name}님!"

if __name__ == "__main__":
    mcp.run(transport="streamable-http")   # → http://<내주소>:8101/mcp

그다음 마켓플레이스에 등록합니다. 🖥️ MCP 서버 페이지의 “새 MCP 서버 등록” 폼을 쓰거나, API로:

curl -X POST {{ public }}/api/servers \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <공유토큰>" \
  -d '{
    "id": "myteam-greet",
    "name": "인사 서버",
    "owner": "4조",
    "namespace": "myteam",
    "endpoint": "http://my-public-host:8101/mcp",
    "description": "이름을 받아 인사"
  }'
endpoint는 게이트웨이(마켓플레이스 서버)가 접속할 수 있는 도달 가능한 주소입니다. 소비자에게 노출되는 주소가 아니라, 마켓플레이스에서 실제로 접속되는 MCP 주소를 적으세요. 보안상 사설/내부망·클라우드 메타데이터(169.254.x) 주소는 거부됩니다(SSRF 방지).

등록하면 마켓플레이스가 즉시 list_tools를 호출해 도구·스키마를 자동 수집합니다. 도구를 바꿨다면 UI의 “도구 새로고침” 또는 POST /api/servers/<id>/refresh.

② 게이트웨이로 사용하기 (소비자)

👤 컨슈머 페이지에서 컨슈머를 만들고, 쓰고 싶은 서버들을 구독 체크하세요. 그러면 아래 한 줄이면 끝입니다.

LangChain 에이전트

from langchain_openai import ChatOpenAI
from langchain_mcp_adapters.client import MultiServerMCPClient
from langchain.agents import create_agent

URL = "{{ public }}/mcp/consumers/my-agent"   # ← 내 컨슈머 엔드포인트 (토큰 불필요)

client = MultiServerMCPClient({
    "market": {"url": URL, "transport": "streamable_http"}
})
tools = await client.get_tools()              # 구독한 모든 서버의 도구 (서버id__도구명)

agent = create_agent(ChatOpenAI(model="gpt-4o-mini"), tools)
res = await agent.ainvoke({"messages": [("user", "철수에게 인사해줘")]})
print(res["messages"][-1].content)

순수 MCP 클라이언트 (SDK)

from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

URL = "{{ public }}/mcp/consumers/my-agent"   # 게이트웨이는 토큰 불필요

async with streamablehttp_client(URL) as (r, w, _):
    async with ClientSession(r, w) as s:
        await s.initialize()
        print([t.name for t in (await s.list_tools()).tools])
        out = await s.call_tool("myteam-greet__greet", {"name": "철수"})
        print(out.content[0].text)
개별 서버만 쓸 땐 {{ public }}/mcp/servers/<서버id>에 붙고, 도구 이름은 네임스페이스 없이 greet 처럼 원래 이름 그대로입니다.

③ 서버 상태와 장애 처리

마켓플레이스는 등록된 서버를 주기적으로 폴링합니다. 서버가 죽어도 등록 정보는 지우지 않고 상태만 바꿉니다.

상태의미
ONLINE최근 응답함 — 정상 중계
UNHEALTHY잠시 응답이 늦음 — 아직 중계 시도
OFFLINE오래 응답 없음 — 호출 시 즉시 SERVER_OFFLINE 반환(타임아웃 X)
ARCHIVED아주 오래 죽어 있음 — 목록에서 흐리게 표시, 메타는 보존

폴링과 별개로, 서버가 직접 생존 신고를 할 수도 있습니다(선택):

curl -X POST {{ public }}/heartbeat \
  -H "Content-Type: application/json" -H "Authorization: Bearer <공유토큰>" \
  -d '{"server_id": "myteam-greet"}'

게이트웨이 에러 코드

호출이 실패하면 게이트웨이는 도구 결과 텍스트로 JSON 에러를 돌려줍니다.

error의미
NOT_SUBSCRIBED그 컨슈머가 해당 서버를 구독하지 않음 → UI에서 구독 체크
SERVER_OFFLINE대상 서버가 OFFLINE/ARCHIVED 상태
UPSTREAM_ERROR서버는 살아있다고 보였으나 실제 호출이 실패
UNKNOWN_SERVER / BAD_TOOL_NAME없는 서버 / 잘못된 도구 이름 형식

그 외 HTTP 레벨: 401 UNAUTHORIZED(토큰 없음/오류), 400 ENDPOINT_REJECTED(SSRF 차단된 endpoint).
